Problem & Solution

The Problem

  • Social media prioritizes large follower counts over meaningful connections
  • Users experience shallow interactions and content overload
  • Research: meaningful relationships limited to 5–15 people (Dunbar's Number)

InnerCircle

  • Enforced friend cap (~25) — must remove a friend to add new one
  • Geospatial sharing — posts pinned on interactive 3D globe
  • Friend-based access control — only see posts from your circle
  • Target: students who want personal, not performative social media

Advanced Features

5 implemented (2 required)

1. Authentication & Authorization

  • Better Auth — session-based, email/password
  • Route protection via proxy.ts
  • Friend-based access control on every API

2. Real-Time Functionality

  • Socket.io on custom HTTP server
  • Live notifications: like, comment, friend request, new post
  • Cross-session post data sync (post:updated broadcast)

3. File Handling & Processing

  • Sharp: resize (1920px), WebP conversion, thumbnails (400×400)
  • Parallel upload to DigitalOcean Spaces (S3-compatible)

4. Advanced State Management

  • Redux Toolkit — postsSlice shared across 4+ components
  • Global state: posts, filters, selected post, pagination
  • Real-time sync via syncPostCounts action

5. Integration with External APIs

  • SightEngine AI — cloud-based image content moderation
  • Sync check on upload: nudity, violence, gore, offensive
  • Auto-delete flagged images from S3 + user-facing rejection UI
  • DigitalOcean Spaces — S3-compatible cloud object storage

Key Technical Decisions

Custom Server + globalThis for Socket.io

  • Next.js API Routes can't handle WebSocket upgrades
  • server.ts wraps Next.js with custom HTTP server
  • Share Socket.io instance via globalThis between server (native Node) and API Routes (webpack-bundled)

Friend Cap: Transaction-Level Enforcement

  • Accept handler uses prisma.$transaction()
  • Prevents race condition: two concurrent accepts exceeding cap

Dynamic Globe Marker Clustering

  • Cluster threshold scales with camera altitude: altitude × 2.2
  • Zoomed out: clusters with count badge; zoomed in: individual pins
  • Low-altitude clusters expand into scrollable list panel

AI Moderation Pipeline

  • Upload → S3 → SightEngine API (sync, 12s timeout)
  • Flagged: auto-delete from S3 + return 422 with categories
  • Fail-open design: if API unavailable, upload proceeds
